# MANAS — Deployment Guide

## Backend → Render

### One-time setup

1. **Push to GitHub**
   ```bash
   git add .
   git commit -m "Initial MANAS build"
   git push origin main
   ```

2. **Create Render Blueprint**
   - Go to [render.com](https://render.com) → New → Blueprint
   - Connect your GitHub repo
   - Render reads `render.yaml` automatically
   - Two resources will be created: `manas-db` (PostgreSQL) and `manas-api` (Node web service)

3. **Wait for first deploy**
   - The build command runs migrations automatically (`prisma migrate deploy`)
   - First deploy takes ~3–5 minutes

4. **Seed the database** (run once)
   - Open Render dashboard → `manas-api` → Shell
   ```bash
   npm run db:seed
   ```

5. **Copy the live URL**
   - Your API lives at: `https://manas-api.onrender.com`
   - Test: `curl https://manas-api.onrender.com/health`

6. **Update mobile app**
   - Set `EXPO_PUBLIC_API_URL` in `mobile/.env.production` to the Render URL
   - The `eas.json` preview/production profiles already include this env var

### Environment variables (managed by Render)

| Variable | How it's set |
|---|---|
| `DATABASE_URL` | Auto-linked from `manas-db` |
| `JWT_SECRET` | Auto-generated by Render (`generateValue: true`) |
| `NODE_ENV` | `production` |
| `PORT` | `4000` |
| `FRONTEND_URL` | Update to your frontend domain |
| `GOOGLE_CLIENT_ID` | Add manually when implementing Google OAuth |

### Free tier notes

- **Cold starts**: The free plan spins down after 15 minutes of inactivity. First request after spin-down takes 30–60 seconds. This is expected.
- **Database**: Render's free PostgreSQL expires after 90 days — upgrade to a paid plan or export data before expiry.
- **Logs**: Render dashboard → `manas-api` → Logs

### Re-deploying

Every push to `main` triggers an automatic redeploy. To manually deploy:
```
Render dashboard → manas-api → Manual Deploy → Deploy latest commit
```

### Running migrations after schema changes

```bash
# Local
cd backend
npx prisma migrate dev --name <migration-name>
git add prisma/migrations
git commit -m "Add migration: <name>"
git push  # Render deploys and runs migrate deploy automatically
```
